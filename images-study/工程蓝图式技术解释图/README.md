## 核心要点
- **把抽象概念机械化**：把模型、系统、流程画成一台机器，读者能直观看到“输入如何变成输出”。
- **用公式框先定义逻辑**：先用一句公式解释核心机制，再用图形把公式拆成输入、处理、输出。
- **管线连接要清楚**：输入模块、中心装置、输出结果必须通过管道或箭头连接，方向不能断。
- **蓝图感靠克制配色**：米白网格纸、深蓝线稿、少量橙色标注，就能形成工程图纸质感。
- **文字必须短而准**：适合放术语、标签、公式和结论，不适合塞大段解释。
## Prompt
```plain text
生成一张横向技术解释信息图，比例 16:9，主题是“把模型看成一个函数”。

主题：
- 把模型看成一个函数

风格：
- 工程蓝图式技术解释图。
- 米白色网格纸背景。
- 深蓝色机械线稿。
- 精密工程图纸质感。
- 少量橙色虚线作为尺寸标注、流向箭头和强调线。
- 整体干净、理性、专业。
- 适合技术课程、AI 原理讲解、系统机制说明。

画面结构：
- 顶部左侧放小编号标签。
- 顶部中央放超大中文标题。
- 标题下方放一个横向公式说明框。
- 中心放一个大型机械装置，装置中间写 LLM。
- 左侧放 5 个输入模块，通过管道连接到中心装置。
- 右侧放 1 个输出模块，通过管道从中心装置连接出去。
- 中心装置周围加橙色虚线尺寸标注，强化工程图纸感。
- 底部放两个对比标签，强调“模型无状态”和“产品可以有状态”。
- 最底部放一句总结性结论。

视觉元素：
- 工程图纸边框。
- 网格纸背景。
- 机械舱体。
- 管道。
- 阀门。
- 仪表盘。
- 螺丝。
- 信号监测小屏。
- 模块标签牌。
- 橙色虚线尺寸标注。
- 蓝色细线描边。

文字内容，保持清晰可读：
- 左上角标签：01 · 模型运行机制
- 主标题：把模型看成一个函数
- 公式框：输出 = 模型（系统提示词 + 工具 + 会话历史 + 其他上下文 + 用户指令）
- 左侧输入模块：系统提示词，工具，会话历史，其他上下文，用户指令
- 中心装置：LLM
- 右侧输出模块：生成结果
- 底部左侧标签：模型无状态
- 底部右侧标签：产品可以有状态
- 底部结论：给什么上下文，就生成什么回答。

约束：
- 整体要像完成度高的技术课程配图，不要像普通 PPT。
- 画面要有工程制图感和机械装置感。
- 输入、处理、输出的方向必须一眼看懂。
- 所有文字都要放在独立容器里。
- 字号要足够大，标签要短。
- 留白要干净，信息丰富但不能拥挤。

严格禁止：
- 禁止写实照片、3D 渲染、赛博朋克、暗黑科技风。
- 禁止复杂彩色渐变、霓虹光效、厚重阴影。
- 禁止把结构画成普通 PPT 流程图或扁平商务插画。
- 禁止输入模块、中心装置、输出模块之间管线断裂、方向反了或连接关系混乱。
- 禁止公式框、标题、标签、管线、机械装置互相遮挡或溢出边框。
- 禁止大段小字，所有文字必须短句、清晰、放在容器内。
```
